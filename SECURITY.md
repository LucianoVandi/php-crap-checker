# Security Policy

## Supported Versions

Only the latest released version of `lvandi/php-crap-checker` receives security fixes.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

You can report a vulnerability in one of two ways:

1. **GitHub Security Advisories** — use the
   [Report a vulnerability](https://github.com/LucianoVandi/php-crap-checker/security/advisories/new)
   link in the Security tab of this repository (preferred).

2. **Email** — send a message to `developer@holisoft.it` with the subject line
   `[php-crap-checker] Security Vulnerability`.

### What to include

- A description of the vulnerability and its potential impact
- Steps to reproduce or a minimal proof-of-concept
- Affected versions (if known)

### Expected response times

| Action | Target time |
|--------|-------------|
| Initial acknowledgement | ≤ 72 hours |
| Triage and severity assessment | ≤ 7 days |
| Fix or mitigation | Depends on severity — critical issues are prioritised |

We will coordinate a disclosure date with you once a fix is ready. Public disclosure
before a fix is available may put users at risk; we ask that you follow responsible
disclosure practices.
